# RAi System
Ryan Candish - Rāi Overclock D0S, everything you never thought of five years ago.

This repository is a multi-page HTML system designed to feel like one connected operating environment rather than a loose set of standalone documents. The pages share a common identity, navigation, and purpose while each offers a different lens into the RAi / RY / RI world.

## Core experience pages
- index.html — the main hub and bootloader-style launcher for the system.
- RAilingua.html — the archive / story experience with a layered, immersive interface.
- ninjastar.html — a workspace page for notes, study prompts, and snapshot export.
- ry-rai-ri.html — a companion page for the RY / RAI / RI conceptual framework.
- rios-v1.html — the original RAiOS v1 experience.
- kurai-edition.html — a minimal, windowed desktop-style environment.
- kali-os.html — a neon terminal/IDE-like experience.
- raios-v4.html — a functional windowed OS shell with files, editor, and terminal.
- rai-archive.html — an archive-style knowledge OS for agents, ideas, and companion interactions.

## How the system works
1. Open index.html to enter the main bootloader and choose a mode.
2. Use the shared navigation on each page to move between the experiences.
3. Each page is intentionally built as a different “kernel” or “mode” of the same larger project.
4. On ninjastar.html, use the snapshot/export action to save a local HTML copy of the current workspace state.

## Notes for GitHub Pages
These files can be served directly from the repository root on GitHub Pages. The simplest setup is to publish the repository as a static site with no build step.
